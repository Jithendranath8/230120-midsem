# Dataset Information

## Dataset: Breast Cancer Wisconsin (Diagnostic)

### Source
Loaded programmatically from `scikit-learn` via `sklearn.datasets.load_breast_cancer()`.
Originally from the UCI Machine Learning Repository, created by Dr. William H. Wolberg at the University of Wisconsin Hospitals, Madison.

### Description
- **Samples**: 569 (455 train / 114 test after 80/20 split)
- **Features**: 30 real-valued features from digitized FNA images of breast masses
- **Target**: Binary — Malignant (0 -> -1) or Benign (1 -> +1)
- **Feature types**: Mean, standard error, and worst values of 10 cell nucleus measurements (radius, texture, perimeter, area, smoothness, compactness, concavity, concave points, symmetry, fractal dimension)

### Preprocessed Files

| File | Description |
|------|-------------|
| `X_train.npy` | Training features (455 x 30), standardized |
| `X_test.npy` | Test features (114 x 30), standardized |
| `y_train.npy` | Training labels in {-1, +1} |
| `y_test.npy` | Test labels in {-1, +1} |
| `sigma.npy` | Gaussian kernel width (5th percentile of pairwise distances) |
| `best_R.npy` | Best norm constraint R=20, selected via 5-fold CV |

### Preprocessing Steps
1. Features standardized to zero mean, unit variance (`StandardScaler`)
2. Labels converted from {0, 1} to {-1, +1} as required by the KLR formulation
3. Gaussian kernel width sigma set to 5th percentile of pairwise Euclidean distances on training set (paper methodology)
4. 80/20 stratified train/test split with `random_state=42`
5. R selected from {1, 5, 10, 20} via 5-fold cross-validation on training set only (no test leakage)

### Usage in Notebooks
- **task_2_1.ipynb**: Generates all `.npy` files above
- **task_2_2.ipynb**: Loads data, runs NC/Margin/Auxiliary, saves `best_R.npy`
- **task_2_3.ipynb**: Loads data + `best_R.npy` for final comparison
- **task_3_1.ipynb**: Loads data + `best_R.npy` for ablation study
- **task_3_2.ipynb**: Generates its own synthetic dataset (does not use these files)

### Why No External Data Files
The dataset is fetched programmatically via scikit-learn, so no manual download is needed. All preprocessed `.npy` files are regenerated by running `task_2_1.ipynb`, ensuring full reproducibility.
